## 1. 产品架构概览

### 1.1 产品本质

「审美提升 + 提示词训练 + AI 诊断」工具站，提供 **判断 → 描述 → 修改** 的完整能力闭环。全程不让用户手绘/做设计，只做判断、描述和练习。

### 1.2 MVP 范围

| 模块 | 优先级 | MVP 是否包含 | 角色定位 |
|------|--------|--------------|----------|
| 模块1 审美判断训练 | P0 | 是 | 学习型，重复性，能力建设 |
| 模块3 AI 设计诊断 | P0 | 是 | 工具型，一次性任务，高频使用 |
| 模块2 提示词训练 | P1 | 否（占位） | — |
| 模块4 知识库 | P1 | 否（占位） | — |
| 模块5 素材库 | P2 | 否（占位） | — |

### 1.3 核心架构原则

- **诊断优先**：立即诊断是首页与导航的第一优先级入口（基于 DAU 1000+ 目标和"上传即用"的工具属性）
- **模块松耦合**：诊断与训练各自独立入口、互不干扰，通过"维度打分数据"形成弱关联桥接
- **数据桥接双载体**：个人中心"能力画像"面板 + 训练页内联提示，共同承载"诊断自测 → 训练侧重"的关联

## 3. 用户流程

### 3.1 核心流程 A：诊断 → 修改 → 迭代

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  上传图片  │ →  │ AI 诊断   │ →  │ 查看结果  │ →  │ 采纳建议  │ →  │ 输入修改  │
│ URL/截图   │    │ (渐进填充) │    │ (标签切换) │    │ /微反馈   │    │ 方案      │
└──────────┘    └──────────┘    └──────────┘    └──────────┘    └────┬─────┘
                                                                        │
                                                                        ▼
                                                              ┌──────────────────┐
                                                              │ AI 预判效果/风险   │
                                                              │ + 优化建议         │
                                                              └────────┬─────────┘
                                                                       │
                                                                       ▼
                                                              ┌──────────────────┐
                                                              │ 生成最终修改提示词 │
                                                              │ → 一键复制带走到   │
                                                              │   外部 AI 工具     │
                                                              └──────────────────┘
```

**关键节点说明**：
- 上传：支持拖拽上传 / 粘贴 URL / 截图粘贴（Ctrl+V）
- 诊断等待：≤10 秒，渐进式填充（"正在分析配色..."→"正在检查对齐..."），非阻断，用户可滚动查看已出部分
- 离开页面：诊断作废，需重新开始
- 结果查看：标签切换（概览/问题/建议/迭代），sticky 底部栏始终显示"复制完整修改提示词"
- 迭代预判：支持多轮迭代

### 3.2 核心流程 B：训练 → 学习 → 应用

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 选择训练  │ →  │ 答题      │ →  │ AI 解析   │ →  │ 手动下一题│
│ 模式      │    │ (选择/打分)│    │ (原地展开) │    │ (消化节奏)│
└──────────┘    └──────────┘    └──────────┘    └──────────┘
                       │
                       ▼
              ┌──────────────────┐
              │ 积累设计词汇      │
              │ 正确率/等级提升    │
              └────────┬─────────┘
                       │
                       ▼
              ┌──────────────────┐
              │ 在诊断中应用所学  │
              │ (维度打分校准)    │
              └──────────────────┘
```

**关键节点说明**：
- 模式选择：顶部标签切换（好坏对比/找茬/维度打分）
- 反馈：原地展开，保留题目对照 + 解析在下方滑出
- 流转：手动点击"下一题"，强调"消化"而非速度
- 维度打分：用户先自评 4 维度（滑块）→ 提交 → AI 评分 → 差异高亮对照

### 3.3 数据桥接流程

```
维度打分自评
    │
    ▼
用户评分 vs AI 评分（4维度）
    │
    ▼
差异数据存储
    │
    ├──→ 个人中心：能力画像面板（维度得分曲线 + 推荐优先练习项）
    └──→ 训练页：顶部内联提示（"基于你的自测，建议优先练习：对比、重复"）
```

### 3.4 新用户首次引导流程

```
注册/登录
    │
    ▼
3 步引导（≤3步，5分钟内上手）
    │
    ├── Step 1: 产品价值说明（判断→描述→修改）
    ├── Step 2: 立即体验诊断（上传示例图）
    └── Step 3: 邀请加入7天训练计划
    │
    ▼
进入首页（已登录态，显示进度/推荐条带）
```

#### 4.2.2 三种模式的内容结构

**好坏对比模式**：
```
QuestionArea
├── ImagePair (ImageA + ImageB)
├── ChoiceButtons (SelectBetterA + SelectBetterB)
└── AnalysisPanel (展开后)
    ├── ResultIndicator (正确/错误)
    ├── GoodDesignAnalysis (好在哪里 + 维度列表)
    ├── BadDesignAnalysis (差在哪里 + 维度列表)
    └── PrinciplesInvolved (涉及设计原则标签)
NextButton
```

**找茬模式**：
```
QuestionArea
├── SingleImage (有缺陷的图)
├── FourOptions (对比/对齐/重复/亲密性 四选一卡片)
└── AnalysisPanel (展开后)
    ├── CorrectAnswer
    ├── DefectAnnotation (图上圈出缺陷位置)
    └── ImprovementSuggestion
NextButton

[多缺陷图片变体：一张图多个问题点，4选项可多选]
```

**维度打分模式**：
```
QuestionArea
├── UploadZone (上传任意设计图)
├── SelfScoringSliders × 4
│   ├── 配色 Slider (1-10)
│   ├── 排版 Slider (1-10)
│   ├── 字体 Slider (1-10)
│   └── 留白 Slider (1-10)
├── SubmitButton
└── ComparisonPanel (提交后)
    ├── DiffHighlight (差异高亮，非柱状图)
    │   示例: "你给配色 8 分，AI 给 5 分，偏差 3"
    ├── AIScoreDetail (每维度详细点评)
    └── ImprovementSuggestion
NextButton
```

#### 4.2.3 建议条带

- 位置：模式标签上方
- 内容："基于你的自测，建议优先练习：[维度1]、[维度2]" + "查看画像"链接
- 无自测数据时：隐藏
- 点击"查看画像"：跳转个人中心能力画像面板

#### 4.2.4 进度统计条

- 位置：页面底部（非 sticky）
- 内容：当前模式进度点 + 正确率 + 完成题数 + 等级
- 难度分级：初级/中级/高级，可在标签旁切换

## 5. 交互模式

### 5.1 诊断等待期交互

| 属性 | 规范 |
|------|------|
| 阻断性 | 非阻断，用户可滚动查看已出部分 |
| 进度展示 | 渐进式步骤列表（已完成✓ / 进行中⏳ / 待处理○） |
| 步骤文案 | "正在分析配色..." → "正在检查对齐..." → ... |
| 离开页面 | 诊断作废，需重新开始 |
| 离开提示 | 底部固定提示"⚠ 离开页面将作废本次诊断" |
| 完成触发 | 全部步骤完成 → 自动切换到结果页"概览"标签 |
| 响应时间 | ≤10 秒（PRD 要求） |

### 5.2 训练答题反馈与流转

| 属性 | 规范 |
|------|------|
| 反馈呈现 | 原地展开，保留题目对照 |
| 展开位置 | 题目区域下方滑出 AnalysisPanel |
| 展开动画 | 高度过渡 + 淡入，300ms |
| 流转方式 | 手动点击"下一题"按钮 |
| 流转节奏 | 强调"消化"而非速度，不自动加载下一题 |
| 下一题位置 | AnalysisPanel 内底部右对齐 |

### 5.3 维度打分对照

| 属性 | 规范 |
|------|------|
| 自评方式 | 4 个滑块（配色/排版/字体/留白），1-10 分 |
| 提交后展示 | 差异高亮（非柱状图） |
| 高亮规则 | 只标出偏差大的维度（偏差 ≥2 触发高亮） |
| 文案格式 | "你给配色 8 分，AI 给 5 分，偏差 3" |
| 目的 | 突出"校准"目的，帮助用户校准审美标尺 |
| 附加信息 | AI 每维度详细点评和改进建议 |

### 5.4 诊断结果页标签切换

| 属性 | 规范 |
|------|------|
| 标签 | 概览 / 问题 / 建议 / 迭代 |
| 切换方式 | 点击标签，内容区原地切换 |
| 默认标签 | 概览 |
| sticky 底部栏 | 始终显示"复制完整修改提示词"，不随标签切换变化 |
| 状态保持 | 切换标签不丢失各标签内的操作状态（如迭代输入内容） |

### 5.5 微反馈机制

| 属性 | 规范 |
|------|------|
| 位置 | 每条建议卡片底部 |
| 形式 | "👍有用" / "👎无用" 两个轻量按钮 |
| 触发后 | 点击的按钮高亮，另一个淡出 |
| 数据用途 | 采集"采纳率"细颗粒度数据（PRD 成功指标：采纳率 ≥70%） |
| 设计要求 | 轻量不抢眼，不干扰阅读 |

### 5.6 迭代预判交互

| 属性 | 规范 |
|------|------|
| 输入 | textarea，文字描述修改方案 |
| 触发 | 点击"AI 预判"按钮 |
| 输出 | 预期效果 + 潜在风险 + 合理性评分(1-10) + 优化建议 |
| 多轮 | 支持，点击"继续迭代"基于上一轮结果继续 |
| 上下文 | 基于原图和修改意见进行预判 |

### 5.7 通用交互反馈

- 所有操作有明确的加载/成功/失败反馈（PRD 6.3）
- 按钮点击：ripple 或 scale 反馈
- 表单校验：即时内联提示
- 复制操作：toast 提示"已复制到剪贴板"

## 6. 状态规范（空/错误/边界）

### 6.1 空状态

统一采用"引导式空状态"——不展示空容器，用插画 + 一句话 + 行动按钮引导首次使用。

| 场景 | 空状态内容 |
|------|------------|
| 能力画像面板（无自测数据） | 插画 + "完成首次维度打分，建立你的能力画像" + [去维度打分] |
| 训练页建议条（无自测依据） | 隐藏（不显示空提示） |
| 诊断历史（空） | 插画 + "上传第一张设计图，开始你的诊断之旅" + [立即诊断] |
| 首页进度条带（未开始训练） | 隐藏（不显示空提示） |

### 6.2 错误状态

| 场景 | 错误状态处理 |
|------|--------------|
| AI 诊断失败 | 卡片式错误（图标 + 说明 + 手动重试按钮），不自动跳转；自动重试 3 次后才展示错误卡片 |
| 图片上传校验失败 | 即时内联提示（格式/大小不符） |
| URL 抓取失败 | 提示"无法获取图片，请改用上传方式" |
| 网络断开 | 全局 toast 提示 + 重试按钮 |
| AI 诊断降级 | MVP 不实现 cached 降级，失败即报错重试 |

## 7. 视觉组织结构

### 7.1 设计风格

基于 PRD 7.1 和参考设计系统（苹果风格）：

- **视觉风格**：简洁专业，体现"审美工具站"的定位
- **配色方案**：中性色（白/灰/黑）为主，点缀低饱和度蓝绿色主色调
- **字体**：无衬线字体为主（Inter / 思源黑体），保证可读性
- **留白**：充足留白，体现"我们懂设计"的自信
- **圆角**：适度圆角，柔和但不失专业感
- **阴影**：克制使用，层级分明

### 7.2 视觉层级

```
层级 1（最高）：主 CTA 按钮（立即诊断）、关键标题
层级 2：次 CTA 按钮（开始训练）、卡片标题、得分数字
层级 3：正文内容、问题描述
层级 4：辅助说明、微反馈按钮、标签
层级 5（最低）：占位模块（"即将上线"）、次要信息
```

### 7.3 内容组织模式

| 模式 | 应用场景 |
|------|----------|
| 标签切换 | 诊断结果页（概览/问题/建议/迭代）、训练页（3种模式） |
| 卡片列表 | 问题列表（按严重度）、建议列表、功能展示 |
| 分栏布局 | 诊断概览（雷达图 + 维度明细）、首页进度条带 |

### 7.4 响应式断点

| 断点 | 宽度 | 布局调整 |
|------|------|----------|
| 桌面 | ≥1200px | 完整布局，分栏并排 |

### 7.5 无障碍

- 符合 WCAG 2.1 AA 标准（PRD 6.3）
- 所有交互元素键盘可达
- 颜色对比度达标
- 图片有 alt 文本
- 表单字段有 label

## 9. 已确认决策汇总

以下决策均经用户确认，作为本规范的约束基础：

### 9.1 架构决策

1. **导航主次**：立即诊断（一级）> 训练（二级），未上线模块显示"即将上线"灰色占位
2. **模块关系**：诊断与训练各自独立入口、互不干扰，通过"维度打分数据"形成弱关联桥接
3. **诊断结果页结构**：标签切换（概览/问题/建议/迭代）+ 底部 sticky"复制修改提示词"操作栏
4. **数据桥接双载体**：个人中心"能力画像"面板 + 训练页内联提示

### 9.2 布局决策

5. **首页 hero**：立即诊断（主CTA）+ 开始训练（次CTA）
6. **首页返回用户**：hero 下方插入轻量"进度/推荐"条带（仅登录用户可见）
7. **训练页组织**：顶部标签切换 3 模式，维度打分标签内含上传区
8. **能力画像位置**：个人中心面板展示维度得分曲线 + 推荐优先练习项

### 9.3 交互决策

9. **诊断等待期**：非阻断渐进式填充，离开页面诊断作废
10. **训练反馈**：原地展开（保留题目对照 + 解析滑出），手动点击"下一题"流转
11. **维度打分对照**：差异高亮（只标出偏差大的维度），突出"校准"目的

### 9.4 状态决策

12. **额度墙**：诊断硬拦截（超出即停）、训练软提示（下一题触发）
13. **AI 诊断降级**：MVP 不实现 cached 降级，失败即报错重试
14. **微反馈**：每条建议带"有用/无用"微反馈按钮，采集采纳率细颗粒度数据

## 一、Demo 简介

### 这是什么

“美商培养工具站”是一款面向设计师、产品经理、独立开发者和 AI 创作者的网页工具。它希望解决一个常见问题：很多人能借助 AI 快速生成页面，却很难判断页面为什么不够好看，也很难把审美判断转化为可以直接执行的修改动作。

### 面向谁

- 使用 AI 生成网页或视觉方案的创作者
- 需要快速检查页面质量的产品经理和运营人员
- 正在学习排版、配色、字体层级和留白的设计初学者
- 想建立稳定审美判断标准的独立开发者

### 核心功能

#### 1. AI 设计诊断

上传 JPG、PNG 或 WebP 页面设计图，选择设计类型和诊断重点，系统通过视觉模型分析：

- 排版与布局
- 配色
- 字体与文字层级
- 留白与视觉平衡

诊断结果包含总体评分、四维评分、问题位置、观察、原因、修改建议、设计原则和可执行 Prompt。

#### 2. 建议执行后的效果图预览

每条修改建议都可以单独生成效果图，支持：

- 查看原图与建议执行后的对比
- 复制可执行 Prompt
- 生成失败后重试
- 图片失效时显示清晰的恢复路径

效果图用于帮助用户理解“建议执行后可能变成什么样”，而不是直接替代人工设计判断。

#### 3. 三种审美训练

- 好坏对比：对同主题的 A/B 设计稿做选择，并获得设计原则解析
- 找茬训练：从页面设计中识别布局、配色、层级、留白等问题
- 维度打分：分别给四个设计维度评分，与参考评分进行对照

训练结果会保存到个人中心，用户可以看到训练次数、诊断次数、综合能力和薄弱维度，并删除不需要的历史记录。

## 二、Demo 创作思路

### 灵感来源

现在 AI 已经可以很快生成一个页面，但“生成出来”不等于“设计合理”。真正影响使用体验的，往往是信息层级、对齐关系、视觉焦点、色彩对比和留白节奏。

我在使用 AI 做前端时经常遇到这样的情况：页面能跑起来，但不知道具体哪里不够好；即使发现问题，也很难把“这里不好看”说成 AI 能准确执行的修改指令。

### 想解决的问题

传统审美学习通常依赖零散案例和主观评价，用户知道结果，却不一定理解原因；普通 AI 对话又容易只给抽象建议，缺少具体位置、修改参数和验证路径。

因此我把产品设计成一个闭环：

> 上传设计图 → 获得结构化诊断 → 复制可执行 Prompt → 查看修改后的效果参考 → 通过训练建立自己的判断标准

### 为什么这样取舍

初赛 Demo 没有追求复杂的用户系统、支付和社交功能，而是优先打通三个真实价值：

1. 让用户知道问题在哪里。
2. 让用户知道为什么是问题。
3. 让用户知道下一步可以怎么改。

训练模块则用于解决另一个问题：用户不能永远依赖 AI，也需要逐步形成自己的视觉判断力。

## 四、TRAE 实践过程

这个 Demo 不是一次生成完成，而是通过 TRAE 按功能拆分、逐步验证和连续修复完成的。

### 阶段 1：从静态设计到可运行前端

- 使用 TRAE 检查原有静态 HTML、设计资源和页面结构。
- 保留原有蓝色视觉语言，将静态页面迁移为 Vite + 原生 JavaScript SPA。
- 建立真实路由：首页、诊断、训练、个人中心、历史记录。
- 验证动态路由刷新不出现服务器 404。

### 阶段 2：打通 AI 设计诊断闭环

- 实现图片选择、拖拽、粘贴和格式校验。
- 增加设计类型、设计目标和诊断重点配置。
- 通过服务端保护 API Key，使用多模态视觉模型分析上传图片。
- 对模型输出增加结构化 JSON 提取和字段校验。
- 针对 API Key 错误、超时、非法 JSON、网络失败实现中文错误状态和重试。

### 阶段 3：从诊断结果到可执行修改

- 为每条诊断问题生成可复制 Prompt。
- 增加建议执行后的效果图生成接口。
- 使用原图 + 修改建议调用生图模型。
- 增加原图/效果图对比弹窗、加载态、失败重试和图片失效处理。
- 优化生图 Prompt，约束保持原布局、文字、品牌和信息结构，减少模型随意重绘。

### 阶段 4：建立审美训练与个人反馈

- 建立 80 道维度训练题库，四个维度各 20 道。
- 实现随机抽题、冷却期、维度去重和题目场景去重。
- 好坏对比支持 AI 独立判断，找茬训练改为多选问题，维度打分记录用户与参考分差异。
- 使用 localStorage 保存训练历史、诊断历史和能力画像。
- 增加个人中心、历史记录、删除记录和蓝色 Apple Glass 动效。

## 五、开发中的问题与解决方式

### 1. 页面曾出现白屏

原因不是业务页面没有渲染，而是页面内容默认使用 `opacity: 0` 等待滚动揭示动画；当图标或 IntersectionObserver 初始化异常时，主体内容没有降级显示。

解决方式：为图标、IntersectionObserver 和动态内容增加错误隔离，并在动画不可用时立即显示内容。

### 2. 上传图片后页面闪烁

原因是每次配置项变化都整体重绘页面，导致图片预览和表单节点被重复替换。

解决方式：将上传页拆成状态化组件，明确区分 idle、loading、success、error，并对 Blob URL 实现所有权转移和幂等释放。

### 3. 视觉模型返回非法 JSON

视觉模型可能返回 Markdown 代码块、解释文字或不完整 JSON。

解决方式：服务端增加 JSON 提取、字段白名单、评分范围校验和问题结构校验；失败时返回可理解的错误状态，不把模型原始异常直接暴露给用户。

### 4. 训练题目重复

初始版本题目数量少，且随机策略只做了简单打乱。

解决方式：扩充到 80 道题，按维度和场景去重，并加入最近题目冷却、连续维度规避和随机抽题。

### 5. 训练页面在不同分辨率下不齐平

解决方式：将图片、选项、统计和解析区域重新组织为稳定网格；提交后保留原图原貌，不再直接在原图上叠加状态标签；统计卡片统一从图片左侧对齐。

## 七、Demo 价值总结

我希望做的不是一个“替用户审美”的黑盒，而是一套帮助用户逐步理解设计、执行修改并形成判断力的工具。

AI 负责观察和解释，人负责选择、验证和学习；诊断解决当下页面的问题，训练帮助用户减少对 AI 的长期依赖。

这也是“美商培养工具站”最核心的价值：

> 让每一次 AI 生成，都不只是得到一个页面，而是更进一步理解什么样的页面值得被留下。

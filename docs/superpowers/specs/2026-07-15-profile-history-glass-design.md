# 个人中心与历史记录蓝色 Apple Glass 设计规范

## 目标

完善个人中心，让用户能看到训练记录、诊断历史和能力分析；所有记录使用 localStorage 保存，并支持查看、单条删除和清空。视觉上保持现有蓝色主调，采用浅色 Apple Glass 风格。

## 页面范围

- `/profile`：个人概览、四维能力分析、训练进度、最近记录、诊断历史。
- `/history`：统一历史记录页，支持按类型筛选、查看和删除。
- `/training`：训练入口和各训练模块入口保持现有功能，完成训练后写入对应历史记录。
- 诊断报告页：诊断成功后写入诊断历史，历史记录可重新打开报告。

## 数据模型

统一使用带版本号的 localStorage 数据结构：

- `meishang.profile.history.v1`
- `meishang.profile.training.v1`
- `meishang.profile.diagnosis.v1`

每条记录至少包含 `id`、`type`、`createdAt`、`title`、`score`、`summary`。读取时必须容错：不存在、损坏或字段不完整的数据应回退为空数组，不得导致白屏。

训练记录分为维度打分、找茬训练和好坏对比；诊断记录保存设计名称、总分、四维分数、provider 和 reportId。删除操作必须只影响对应记录，不得清除其他类型数据。

## 个人中心信息架构

1. 顶部概览卡：训练总次数、诊断总次数、综合能力分、最近活动时间。
2. 四维能力分析：排版与布局、配色、字体与文字层级、留白与视觉平衡；使用横向进度条或环形分数，标记最薄弱维度。
3. 训练进度：三种训练类型分别展示次数、平均分和最近一次成绩。
4. 最近记录：最多展示 6 条，支持查看和删除；提供“查看全部”入口。
5. 诊断历史：展示最近诊断，支持查看报告、删除；没有数据时显示“立即诊断”引导。

## 视觉规范

- 页面主背景保持浅蓝渐变，不改为黑色主题。
- 主色使用深青蓝/苹果蓝；成功使用绿色，错误使用红色，警告使用橙色，均保持低透明度背景。
- 卡片使用半透明白色背景、`backdrop-filter: blur(18px)`、浅蓝边框和柔和蓝色阴影。
- 重点卡片使用 20–28px 圆角，普通列表卡片使用 14–18px 圆角。
- 内容最大宽度保持 1180–1280px，桌面端采用两列或三列网格，移动端单列堆叠。
- 统一按钮高度至少 44px，删除操作使用低强调度按钮，避免误触。

## 动画规范

- 页面内容进入：opacity 0→1、translateY 12px→0，180–260ms。
- 卡片列表按 30–45ms 间隔错峰进入，最多延迟 240ms。
- 弹窗：背景遮罩淡入并轻微模糊，卡片 translateY 18px→0、scale 0.98→1，220ms。
- 删除：记录先淡出并缩小，再从 DOM 移除，180ms；删除失败时恢复原卡片并显示错误提示。
- 页面切换保持现有 SPA 路由，不强行拦截浏览器返回；切换时只做内容容器淡入淡出。
- `prefers-reduced-motion: reduce` 时关闭位移、缩放和错峰动画，只保留短淡入淡出。

## 交互与错误处理

- 查看记录进入对应报告/结果页；记录不存在时显示“记录已删除或已过期”，提供返回入口。
- 单条删除弹出蓝灰玻璃确认框，明确显示记录标题；确认后更新 localStorage 和页面。
- 清空全部必须二次确认，并明确不可恢复。
- localStorage 写入失败时保留当前页面状态，并显示“本次记录未能保存”，不可白屏。
- 所有空状态都提供下一步操作，不显示空白区域。

## 验收标准

- 刷新 `/profile` 后历史数据仍存在。
- 三种训练记录和诊断记录互不覆盖。
- 个人中心统计值与历史记录实时一致。
- 单条删除、清空全部、取消删除均可用。
- 诊断历史可以重新打开报告；不存在的记录有兜底页。
- 桌面端卡片对齐，移动端不横向溢出。
- 页面切换、弹窗、卡片删除动画自然，且减少动画设置生效。
- `npm run build` 和 `npm test` 全部通过。
